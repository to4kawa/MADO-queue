# Review request: MADO hub sample data

## 背景

MADO hub / form / care / move では、住民記録システム等から切り出したコピーDBを参照する可能性があります。

ただし、実在する住民情報を匿名化・サニタイズして外部レビューに出す方法は、再識別リスクや確認コストが高くなります。

そこで、住民記録システム標準仕様書等の公開仕様を参照し、完全架空の検証用データセットを作る方が、OSSとして議論しやすいのではないか、という提案です。

## 提案

実データを加工するのではなく、最初から架空の住民、世帯、住所、異動履歴、手続きケースを作ります。

このディレクトリには、以下を置いています。

- `schema.sql`: MADO hub 検証用の仮スキーマ
- `seed.sql`: 完全架空のサンプルデータ
- `README.md`: 目的、前提、レビュー観点

## 確認いただきたい点

1. この粒度で MADO hub / form / care / move の検証に使えそうか
2. 個人、世帯、住所、異動履歴、手続きケースの分け方が妥当か
3. MADO 側で実際に必要になりそうな項目が不足しているか
4. 公開ダミーデータであっても持たせない方がよい項目があるか
5. hub 公開後に、正式仕様へ寄せる場合の修正方向は何か

## 明確にしたい範囲外

- 住基ネット接続仕様の再現
- 住民記録システム全体の再実装
- 実データの匿名化手順
- 本番データ移行仕様
- MADO hub の正式スキーマ確定

## 期待する効果

- 個人情報を含めずに外部レビューできる
- UI、帳票転記、検索、テスト設計を早期に検討できる
- hub 公開前に、必要項目と不要項目の線引きを議論できる
- 他自治体や外部ベンダーが、実データなしで検証に参加しやすくなる
