# MADO hub sample data proposal

このディレクトリは、MADO hub / form / care / move の公開前に、外部レビュー可能な検証用データ構造を議論するためのたたき台です。

## 目的

- 実在する住民情報を使わず、完全架空データだけでレビュー可能にする
- 実データの匿名化・サニタイズに伴う再識別リスクを避ける
- 住民記録システム標準仕様書等の公開仕様を参照しつつ、MADO が必要とする最小コピーDBの粒度を議論する
- hub 公開前でも、検索、世帯選択、申請書転記、異動履歴参照、テスト設計の論点を整理する

## 前提

この案は本番接続仕様ではありません。

- 住基ネットに接続しません
- 実在する住民情報を含みません
- 実在する住所、氏名、世帯構成を再現しません
- 標準準拠システムそのものを再実装しません
- MADO hub の正式スキーマではありません

この案は、住民記録システム標準仕様書やデータ要件・連携要件の考え方を参考にした、OSSレビュー用の仮スキーマです。

## 参照する公開情報

- デジタル庁「地方公共団体の基幹業務システムの統一・標準化」
- 住民記録システム標準仕様書（総務省）
- データ要件・連携要件の標準
- MADO README に記載された hub / form / care / move の構成方針

## この案で扱う最小単位

MADO の窓口支援で参照される可能性が高い最小単位として、以下だけを仮置きします。

| 領域 | 仮テーブル | 目的 |
|---|---|---|
| 個人 | `sample_residents` | 氏名、カナ、生年月日、性別等の参照 |
| 世帯 | `sample_households` | 世帯番号、世帯主、住所との関連 |
| 世帯員 | `sample_household_members` | 世帯内続柄、世帯所属期間 |
| 住所 | `sample_addresses` | 住所表記、方書、自治体内住所の確認 |
| 異動 | `sample_resident_movements` | 転入、転居、転出、死亡などの履歴確認 |
| 手続きケース | `sample_procedure_cases` | form / care / move 側のテストシナリオ |

## レビューしてほしい観点

1. MADO のコピーDBとして、この粒度は大きすぎるか、小さすぎるか
2. 個人、世帯、住所、異動履歴を分ける方針でよいか
3. 窓口で申請書に転記するために必要な項目が足りているか
4. 公開ダミーデータとして持たせない方がよい項目があるか
5. hub 公開時に、どの項目を正式仕様側へ寄せるべきか

## 方針

実データをサニタイズしてレビューに出すのではなく、最初から架空データを作ります。

これにより、外部コントリビューターは個人情報を扱わずに、DB設計、UI、帳票転記、検索、テストコードを検討できます。
